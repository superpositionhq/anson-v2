# iMessage on macOS

Read-only access to your local iMessage history (`~/Library/Messages/chat.db`).

## Skip this entire section if

- You're not on macOS
- You don't use iMessage on this Mac
- You don't want the agent to surface specific iMessage chats

## Grant Full Disk Access

The OS sandboxes `chat.db`. Your terminal needs Full Disk Access.

1. Open **System Settings → Privacy & Security → Full Disk Access**.
2. Click **+** and add your terminal app:
   - Apple Terminal: `/System/Applications/Utilities/Terminal.app`
   - iTerm2: `/Applications/iTerm.app`
   - Wezterm / Kitty / etc.: same idea
3. **Quit and relaunch** the terminal app. (Settings change is not picked up until restart.)

## Verify

```bash
bash scripts/verify_imessage.sh
```

Output: `verify_imessage: ok (N chats accessible)`.

## Configure critical chats

Anson asks during Stage B which iMessage chats matter (e.g. "investor group chat — chat ID 99"). Find chat IDs:

```bash
sqlite3 ~/Library/Messages/chat.db \
  "SELECT chat.ROWID, chat.display_name, group_concat(handle.id) FROM chat
   LEFT JOIN chat_handle_join ON chat.ROWID = chat_handle_join.chat_id
   LEFT JOIN handle ON chat_handle_join.handle_id = handle.ROWID
   GROUP BY chat.ROWID;" | head
```

Note: contacts often appear by phone number, not name. Match against your Contacts app to identify.

## Tools

If you have a higher-level `imsg` CLI installed (some macOS toolkits ship one):

```bash
imsg list-chats
imsg read --chat <ROWID> --limit 20
```

If you don't, plain `sqlite3` queries work. Skills generated by anson use whichever is available.

## Sending

Out of scope by default. The agent **reads** iMessage; sending requires AppleScript bridges that anson doesn't install. Reply directly in the Messages app.
